Observações:
	* Na classe "Utilidades", método "readDocument" confere quem a chamou,
	autorizando sua execução somente para o método "main".
		 
	* Na classe "Utilidades", método "sortMap" recebe um objeto como atributo e, 
	confere o tipo de objeto e faz o correto tratamento.
	
	* O codigo foi desenvolvido com base no paradigma Introspective designado
	pelo professor em sala de aula. Para realizar a aplicação deste no 
	algoritmo de TermFrequency foi escolhida a linguagem Java. Por meio da
	Reflection API e acessando a StackTrace o grupo conseguiu realizar uma
	simples aplicação do Introspection no código.
	
	* Introspection é a habilidade de uma função obter informações de si mesma
	em tempo de execução.
	
	* Reflection é a capacidade de acessar e modificar objetos em tempo de
	execução. Além disso, em Java, essa API permite invocar métodos de um
	objeto, mudar acessibilidade, etc.
